# 플러터 면접 스터디

- 0203: 코루틴(Coroutine)이란 무엇이며, 어떤 상황에서 유용하게 사용될 수 있나요?

1. 비동기 프로그래밍 : 프로그램의 작업이 완료될 때까지 기다리지 않고, 다른 작업을 병렬로 처리하는 기법

장점
- 동일한 시간 내에 더 많은 작업을 처리할 수 있어 성능이 향상
- 사용자 요청에 대한 응답 시간이 줄어들어, 시스템의 효율성과 반응 속도가 높아짐
단점 : 코드의 흐름이 직관적이지 않아 디버깅과 유지보수가 어려울 수 있음

2. Coroutine
- co-(함께)와 routine의 합성어로, 비동기 프로그래밍과 동시성(concurrency) 처리를 위한 프로그래밍 구조
- 스레드라는 비싼 자원을 효율적으로 쓰기 위해, 작업자가 쉬는 시간 없이 계속 다른 일감을 던져주는 똑똑한 스케줄링 기술

- 함수나 루틴의 실행을 중단하고 필요 시 다시 재개할 수 있는 특성을 가짐
- 코루틴은 운영체제 스레드와 달리 가벼운 단위로 동작하며, 하나의 스레드에서 다수의 코루틴을 실행할 수 있음

how? 스레드에서 작업의 주체를 변경 -> OS 컨텍스트 스위칭보다 비용이 쌈

3. Thread와의 관계
- 기본적으로 서로 다른 동작 방식을 가짐
- 서로 대체되는 개념보단, 상호 보완적으로 사용됨

차이점
- 앱 레벨 실행 흐름 관리 vs OS 레벨 실행 흐름 관리
- 단일 스레드 내 동작 vs 다중 코어를 활용한 병렬 실행

4. 어떤 상황에서 유용하게 쓰이나?
- 네트워크 및 DB 작업 (I/O Bound)
- 부드러운 UI 환경 유지 (Android, iOS, FE)
- 대규모 동시성 처리 (High Concurrency)
