## 0204 : Dart에서 가비지 컬렉터는 어떤 방식으로 작동하나요? 

### 1. 가비지 컬렉터란?
-  자동 메모리 관리 시스템, 사용되지 않는 객체를 식별하고 힙 메모리에서 해제
- 메모리 누수 방지 효과

### 2. Dart GC 동작방식
#### Young Space Scavenger
-짧은 수명을 가진 객체에 대한 메모리 해제
-객체들이 할당되는 활성화 영역 반, 현재 사용되지 않는 비활성화 영역 반
-활성화 영역이 가득차면 객체의 참조 상황을 보며 정리

#### Parallel Marking and Concurrent Sweeping
- Parallel Marking 단계 : 메모리에서 사용 중인 객체 식별
모든 객체를 순환하며 참조를 기준으로 객체 그래프를 생성
객체그래프에서 사용중인 객체에 Marking
- Concurrent Sweeping 단계 : 객체 그래프를 기준으로 사용되지 않는 메모리 해제
Mark가 없는 객체들은 힙 영역으로 보내지며 다음 GC가 실행될때 재사용

### 3. Dart GC 특징
- flutter와 연동되어 Idle 상태를 감지해 GC를 실행
- 앱의 실행 중지 없이 가비지 컬렉션 수행 가능 => 앱이 사용자와 상호작용하지 않을 때 작업 처리
- hook을 통해 앱의 유휴 상태 파악
