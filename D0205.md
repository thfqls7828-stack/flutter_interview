## Isolate와 스레드의 차이는 무엇인가요? 

###  1. Isolate
정의 : Dart 프로그램에서 실행되는 고유 작업 단위
목적 : 멀티쓰레드의 복잡성과 동기화 문제를 피하면서, 안전한 병렬 처리의 이점을 제공하기 위해 설계
특징
- 개별 메모리 및 스레드 (공유 x)
- 메세지 파싱을 통해 isolate 간 통신 -> 복사본을 보내거나 소유권을 넘김
- CPU 집약적 작업

### 2. Thread
정의 : CPU에 작업을 요청하는 실행 단위
목적 : 메모리 절약 -> 경량 프로세스
특징
- Stack 영역만 새롭게 생성, 힙과 데이터 영역은 공유
- 컨텍스트 스위칭(프로세스 교체 작업)에 비해 오버헤드가 적다.
- 오류 발생 시 프로세스 전체에 영향

### 3. Dart, 단일 쓰레드
- flutter는 60 - 120 프레임 구현 목표 -> 메인 스레드에서 CPU 집중 작업을 하지 않게 하기 위해서
- Dart는 단일 쓰레드 위에서 동작, 복잡하거나 큰 JSON 데이터를 파싱하는 등의 상황에서 UI가 멈출 수 있음(Jank)
